# Pixel Planes — Overview

A browser-based airplane dogfighting game built for learning. Fly with momentum (like the old game Sopwith), shoot and dodge, and play solo or online.

## What the game is

- **Single-player**: You fly against computer-controlled bots. Control your plane with the arrow keys and manage throttle and nose angle. Gravity always pulls down — you need gas to stay up.
- **Split-screen 2-player**: Two planes on the same screen with different controls (player 2 uses WASD).
- **Online multiplayer**: One shared world running on the server — anyone can join, see each other fly in real-time, text chat, or use voice chat over WebRTC.

## The one-address architecture

Everything lives on **one Railway server** at `https://pixel-planes-bryding-production.up.railway.app`:

```
┌─────────────────────────────────────────────────────────────┐
│                   Railway Service                           │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  [Node.js server/server.js]                                 │
│  • Serves all game files (repo root is web root)             │
│  • Runs the live shared world (one big sky, always on)       │
│  • Relays player planes, bullets, chat, voice signaling      │
│                                                               │
│  GET /          →  index.html (the CHOOSER: pick a game)     │
│  GET /solo/     →  solo/index.html (single-player game)      │
│  GET /online/   →  online/index.html (multiplayer)           │
│  WS / WSS (port 8080)     →  Live world state                │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

**A chooser front door, two games, one address:**

0. **The chooser** (`/` — `index.html`) — a simple "How do you want to
   play?" page with two big buttons: 🤖 Single Player and 🌍 Multiplayer
   Online. It uses the root `css/` folder for its styling.

1. **Single-player** (`/solo/` — `solo/index.html`)
   - Play offline by double-clicking `solo/index.html` or via the server
   - Split-screen 2-player and Mouse Mode also live here
   - Controlled by `solo/js/config.js` (single-player tuning numbers)
   - Server does NOT read single-player config

2. **Online multiplayer** (`/online/` — `online/index.html`)
   - One always-on shared world for everyone
   - Text chat, voice chat (WebRTC), player names, server bots to fill empty slots
   - Controlled by `online/js/config.js` (shared with the server so numbers never drift)
   - Server also reads `online/js/bot-ai.js` to run backfill bots

| `tools/` | Testing scripts and small utilities |
| `.specify/` | Old spec infrastructure; inert |

### Top-level files

| File | Purpose |
|------|---------|
| `index.html` | The chooser front page ("How do you want to play?") |
| `package.json` | Node dependencies (`ws` for WebSockets) |

### Single-player code (`solo/js/`)

| File | What it does |
|------|------------|
| `config.js` | All the tuning numbers: speed, gravity, colors, world size. **Start here if you want to experiment.** |
| `game.js` | The main game loop: sets up the canvas, runs the game ~60 times a second, draws everything. |
| `input.js` | Reads the keyboard (arrow keys, spacebar, etc.). Two-player mode reads WASD + regular keys. |
| `plane.js` | The player's airplane: position, velocity, nose angle, drawing. Uses physics from `physics.js`. |
| `enemy.js` | A computer bot plane. Has its own "brain" for steering and shooting. Each bot has its own color and flying style. |
| `bullet.js` | A little glowing projectile shot by planes. Flies in a straight line. |
| `missile.js` | A homing rocket that chases the closest enemy. Runs out of fuel. |
| `pilot.js` | Eject and parachute: press C to eject from your plane, press again to toggle the parachute open/closed. |
| `powerup.js` | Floating bubbles: shield (blue, invincible for a bit), turret (gold, spread shots), skull (red, bad). |
| `explosion.js` | Sparks and fire rings when planes crash or die. |
| `scenery.js` | Background art: clouds, hills, treeline, trees, bushes, haybales, barns. Scrolls with the camera. |
| `sprites.js` | Pixel-art biplane sprites, drawn once and cached for speed. |
| `audio.js` | All sound effects, generated on-the-fly with the Web Audio API (no files needed). Master volume. |
| `physics.js` | Shared helpers for world wrapping: the world is a horizontal loop (fly off the right edge, come out the left). |
| `net.js` | Old lobby code (mostly retired). Single-player ignores it. |
| `uiscroll.js` | Makes menus scrollable on small screens. |

### Online multiplayer code (`online/js/`)

The online client has its own copies of most single-player files (so it can run independently if needed) plus a few extras:

| File | What it does |
|------|------------|
| `config.js` | **Shared with the server.** Tuning numbers like `TARGET_POPULATION` (how many bots to add). The server reads this to match numbers with the client. |
| `bot-ai.js` | **Shared with the server.** The brain that backfill bots use to fly and shoot. Kept in one place so server bots fly exactly like client bots. |
| `game.js` | The online game loop: connects to the server, watches for other players' planes, draws them all. |
| `net.js` | Connects to the WebSocket server, sends/receives player state and chat messages. |
| `voice.js` | Live voice chat over WebRTC (peer-to-peer audio mesh). Handles mic access, connection setup, and peer management. |
| `input.js`, `plane.js`, `bullet.js`, etc. | Same as single-player but tuned for the shared world (online planes don't have split-screen or pause). |

### Server code (`server/`)

| File | What it does |
|------|------------|
| `server.js` | The main server: opens a WebSocket, hands out game files, connects players, relays state, broadcasts world updates. |
| `world.js` | The shared-world state: tracks all players, their planes, bullets, explosions. Runs each frame. |
| `rules.js` | Game rules: can a bullet hit that plane? Does a crash kill you? How much damage? |
| `checks.js` | Small helpers to check if bots or bullets are still alive. |
| `validate-quickstart.js` | A test script to verify the server boots cleanly. |

## How data flows

**Single-player:**
```
Keyboard → input.js → game.js → plane.js (+ enemy.js, bullets, etc.)
                   ↓
              canvas → browser screen
```

**Online multiplayer:**
```
Your keyboard → input.js → net.js → WebSocket → server/world.js
                        ↓
                   game.js draws you + other players' planes
                        ↓
            server broadcasts state → all connected clients
```

## Game loop (roughly 60 times per second)

1. Read input (keyboard, mouse, voice).
2. Update the player's plane position, velocity, angle.
3. Move all bullets and missiles.
4. Check collisions (bullets hit planes, planes hit ground, etc.).
5. Draw the camera view: scenery, planes, bullets, HUD.
6. (Online) Send/receive world updates to/from the server.

## How to run it

**Single-player (offline):**
- Double-click `solo/index.html` in the repo folder.
- Or run `python3 -m http.server` and open `http://localhost:8000/solo/`.

**Everything (via local server):**
- Run `node server/server.js` from the repo folder.
- Open `http://localhost:8080` — you'll get the chooser, same as online.

**Online (same WiFi):**
- Run `node server/server.js`.
- On this computer, open `http://localhost:8080/online/`.
- On another computer on the same WiFi, open `http://[YOUR-IP]:8080/online/` (the server prints the IP).

**Online (public — deployed):**
- The game is live at `https://pixel-planes-bryding-production.up.railway.app/online/`.

## Deploy to Railway

Just push to the `main` branch on GitHub. Railway auto-deploys (see `railway.json` for build config). The server starts with `node server/server.js`.
